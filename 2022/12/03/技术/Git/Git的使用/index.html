Git概述

Git是一个分布式版本控制工具，主要用于管理软件开发过程中的源代码文件。

通过Git仓库来存储和管理这些文件，Git仓库分为两种：

  • 本地仓库：开发人员自己电脑上的Git仓库
  • 远程仓库：远程服务器上的Git仓库

“分布式”主要就体现在这两种不同的仓库

  • commit：提交，将本地文件和版本信息保存到本地仓库
  • push：推送，将本地仓库文件和版本信息上传到远程仓库
  • pull：拉取，将远程仓库文件和版本信息下载到本地仓库

Git安装成功后，在任意目录，右键菜单上多出了两项与Git有关的选项：

  • Git GUI Here：打开Git图形界面
  • Git Bash Here：打开Git命令行

Git代码托管服务

常用的Git代码托管服务

可以用互联网上提供的一些代码托管服务实现Git远程仓库：

  • GitHub
  • 码云
  • GitLab
  • BitBucket

企业开发时也可能会使用企业自己架设的私有代码托管服务。

使用码云代码托管服务

每一个Git远程仓库都会对应一个网络地址，点击【克隆/下载】按钮，在弹出窗口点击【复制】按钮获得网络地址。

为了多人开发，使多个参与者都有权操作远程仓库，需要邀请项目参与者成为当前仓库的成员。

Git常用命令

Git全局设置

安装Git后首先需要设置用户名和email地址，因为每次Git提交都会使用用户信息

  • 设置用户信息
    • 设置用户名：git config --global user.name "<name>"
    • 设置邮箱：git config --global user.email "<email>"
    • 此处的用户名和邮箱与代码托管平台的账号无关，可以任意设置
  • 查看配置信息：git config --list

获取Git仓库

要使用Git对代码进行版本控制，首先需要获得Git仓库。

获取Git仓库有两种方式：

  • 在本地初始化一个Git仓库（不常用）
  • 从远程仓库克隆（常用）

在本地初始化Git仓库

  1. 在任意目录下创建一个空目录，作为本地Git仓库
  2. 进入目录中，右键选择Git Bash Here
  3. 执行命令git init

从远程仓库克隆

可以通过Git提供的命令从远程仓库进行克隆，将远程仓库克隆到本地：git clone <url>

如果当前目录下出现隐藏文件夹.git，则说明Git仓库创建成功。

一个文件夹只能对应一个仓库，且仓库不能嵌套。

工作区、暂存区、版本库

  • 版本库：就是.git隐藏文件夹，里面存储了很多配置信息、日志信息和文件版本信息等
  • 工作区：包含.git文件夹的目录，也称工作目录，主要存放开发的代码
  • 暂存区：.git文件夹中的index文件，也称stage，是一个临时保存文件的地方

Git工作区中文件的状态

Git工作区中的文件存在两种状态：

  • 未跟踪(untracked)，未被纳入版本控制
  • 已跟踪(tracked)，被纳入版本控制
    • 未修改状态(Unmodified)
    • 已修改状态(Modified)
    • 已暂存状态(Staged)

这些文件的状态会随着执行Git的命令发生变化。

本地仓库操作

  • 查看文件状态：git status
  • 将文件的修改加入暂存区：git add
    • 将所有文件的修改加入暂存区：git add *
    • 将指定文件的修改加入暂存区：git add <file-name>
  • 将暂存区的文件取消暂存或是切换到指定版本：git reset
    • 强制回退：git reset --hard
  • 将暂存区的文件修改提交到版本库：git commit
    • 添加对本次修改的描述：git commit -m "<description>"
    • 将指定暂存区文件修改提交到版本库：git commit <file-name>
  • 查看日志：git log

远程仓库操作

  • 查看远程仓库：git remote
    • 它会列出每一个远程服务器的简写，如果已经克隆了远程仓库，那么至少能看到origin，这是Git克隆的仓库服务器的默认名字。
    • 显示远程仓库网络地址：git remote -v
  • 添加远程仓库：git remote add <shortname> <url>
    • 添加一个新的远程Git仓库，同时指定一个可以引用的简写
  • 从远程仓库克隆：git clone <url>
    • 获得一份已经存在了的Git远程仓库的拷贝
    • Git将克隆该Git仓库服务器上的几乎所有数据，包括日志信息、历史记录等，而不仅仅是复制工作所需要的文件
  • 从远程仓库拉取：git pull <remote-name> <branch-name>
    • 从远程仓库获取最新版本合并到当前仓库
    • 如果当前本地仓库不是从远程仓库克隆，而是本地创建的仓库，且仓库中存在文件，此时再从远程仓库拉取文件时会失败并报错fatal: refusing to merge unrelated histories
      • git pull <short-name> <branch-name> --allow-unrelated-histroies可以解决
  • 推送到远程仓库：git push <remote-name> <branch-name>
    • 更改要先推送到本地仓库，再推送到远程仓库
    • 第一次将本地文件推送到远程仓库时，需要进行身份验证（用户名和密码）

分支操作

分支是Git使用过程中的重要概念，可以把工作从开发主线上分离开来，以免影响开发主线。

同一个仓库可以有多个分支，各个分支相互独立，互不干扰。

通过git init创建本地仓库时默认会创建一个master分支。

  • 查看本地分支：git branch
    • 查看远程分支：git branch -r
    • 查看本地分支和远程分支：git branch -a
    • 远程仓库的HEAD分支相当于一个指针，它指向正在操作的分支
    • 绿色带星号的分支表示当前分支
  • 创建分支：git branch <branch-name>
  • 切换分支：git checkout <branch-name>
  • 推送至远程仓库分支：git push <remote-name> <branch-name>
  • 合并分支：git merge <branch-name>
    • 当两个分支相同文件的内容有冲突时，会合并失败，显示冲突文件，并报错Automatic merge falied: fix conflicts and then commit the result
    • 此后再查看冲突文件内容，会出现git对内容修改的指引
    • 冲突文件编辑完成后，使用git add将冲突文件存入暂存区，再使用git commit -i提交即可

标签操作

Git标签指某个分支某个特定时间点的状态。

通过标签，可以很方便地切换到标记时的状态。

人们一般会使用这个功能来标记发布节点（v1.0、v1.2等）

  • 列出已有的标签：git tag
  • 创建标签：git tag <tag-name>
  • 将标签推送至远程仓库：git push <remote-name> <tag-name>
  • 检出标签：git checkout -b <branch-name> <tag-name>
    • 检出标签时需要新建一个分支来指向某个标签

标签是一个静态的概念，定格在Git仓库的某个状态；而分支是一个动态的概念。

在IDEA中使用Git

在IDEA中配置Git

在IDEA中使用Git，本质上还是使用本地安装的Git软件，所以需要在IDEA中配置Git（IDEA可以自动检测Git安装目录）。

获取IDEA仓库

在IDEA中使用Git获取仓库：

  • 本地初始化仓库（VCS菜单选择创建Git仓库）
  • 从远程仓库克隆（VCS菜单选择从版本控制中获取）

.gitignore文件

Git仓库特有的文件，定义当前仓库不需要Git管理的文件/目录。

本地仓库操作

  • 将文件加入暂存区
    • 创建一个新文件时，IDEA会自动询问是否将文件加入暂存区
    • 右键文件名，选择Git→添加也可以将文件加入暂存区
  • 将暂存区的文件提交到版本库
    • 右键文件名，选择Git→提交文件勾选需提交的文件
    • 右键仓库根目录，选择Git→提交目录提交库内所有暂存文件
    • 上方Git按钮中的提交（对勾）
  • 查看日志：上方Git按钮中的显示历史记录（时钟）

在IDEA中，文件名可能有三种颜色

  • 红色：有更改，未被加入暂存区
  • 绿色：有更改，已被加入暂存区
  • 黑色：没有更改，已被纳入Git版本管理

远程仓库操作

  • 查看远程仓库：右键仓库，选择Git->管理远程
  • 添加远程仓库：查看时弹出的窗口中点加号可以添加，点减号则删除
  • 推送至远程仓库
    • 右键仓库，选择Git→推送
    • 上方Git按钮中的提交（对勾），点开后选择提交并推送
    • 上方Git按钮中的推送（绿色箭头）
  • 从远程仓库拉取
    • 右键仓库，选择Git→拉取
    • 上方Git按钮中的拉去（蓝色箭头）

分支操作

  • 查看分支
    • 右键仓库，选择Git->分支
    • 整个IDEA窗口的右下角
  • 创建分支：窗口中点新分支
  • 切换分支：窗口中点击指定分支→签出
  • 将分支推送到远程仓库：窗口中点击指定分支→推送
  • 合并分支：窗口中点击指定分支→将xxx合并到xxx
