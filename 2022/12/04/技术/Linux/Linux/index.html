Linux目录

Linux目录结构

  • /是所有目录的顶点
  • 目录结构呈树形结构

Linux目录介绍

  • bin：存放二进制可执行文件
  • boot：存放系统引导时使用的各种文件
  • dev：存放设备文件
  • etc：存放系统配置文件
  • home：存放系统用户的文件
  • lib：存放程序运行所需的共享库和内核模块
  • opt：额外安装的可选应用程序包所放置的位置
  • root：超级用户目录
  • sbin：存放二进制可执行文件，只有root用户才能访问
  • tmp：存放临时文件
  • usr：存放系统应用程序
  • var：存放运行时需要改变数据的文件，例如日志文件

Linux命令

Linux命令初体验

几个常用命令

Linux命令使用技巧

  • Tab键自动补全
  • 连按两次Tab键，给出操作提示
  • 使用上下箭头快速调出曾经使用过的命令
  • 使用clear命令或Ctrl+l快捷键实现清屏

Linux命令格式

  • command：命令名
  • [-options]：选项，可用来对命令进行控制，也可以省略
  • [parameter]：传给命令的参数，可以是零个、一个或多个
  • 注意：
    • []代表可选
    • 命令名、选项、参数之间有空格进行分隔

文件目录操作命令

ls

  • 作用：显示指定目录下的内容
  • 语法：ls [-al] [dir-name]
  • 说明
    • -a选项：显示所有文件及目录（.开头的隐藏文件也会列出）
    • -l选项：除文件名称外，同时将文件型态（d表示目录、-表示文件）、权限、拥有者、文件大小等详情信息列出
  • 注意：命令ll为命令ls -l的简写形式

cd

  • 作用：切换当前工作目录
  • 语法：cd [dir-name]
  • 说明
    • 用户的home目录：~
    • 目前所在的目录：.
    • 目前目录的上级目录：…
  • 举例
    • 切换到当前目录的上级目录：cd ..
    • 切换到用户的home目录：cd ~
    • 切换到/usr/local目录：cd /usr/local

cat

  • 作用：显示文件内容
  • 语法：cat [-n] file-name
  • 说明
    • -n选项：显示由1开始对所有输出的行数编号
  • 举例
    • 查看/etc目录下的profile文件内容：cat /etc/profile

more

  • 作用：以分页的形式显示文件内容
  • 语法：more file-name
  • 说明：
    • 回车键：向下滚动一行
    • 空格键：向下滚动一屏
    • b：返回上一屏
    • q或Ctrl+C：退出more
  • 举例：
    • 以分页方式显示/etc目录下的profile文件内容：more /etc/profile

tail

  • 作用：查看文件末尾的内容
  • 语法：tail [-f] file-name
  • 说明：
    • -f选项：动态读取文件末尾内容并显示，通常用于日志文件的内容输出，使用Ctrl+C退出查看
    • -数字选项：读取末尾指定行数的内容，默认为10行
  • 举例
    • 显示/etc目录下的profile文件末尾10行的内容：tail /etc/profile
    • 显示/etc目录下的profile文件末尾20行的内容：tail -20 /etc/profile
    • 动态读取/foo目录下的my.log文件末尾内容并显示：tail -f foo/my.log

mkdir

  • 作用：创建目录
  • 语法：mkdir [-p] dir-name
  • 说明
    • -p选项：确保目录文件存在，不存在则创建，通过此选项可实现多层目录同时创建
  • 举例
    • 在当前目录下建立一个名为foo的子目录：mkdir foo
    • 在工作目录下的foo目录中建立一个名为test的子目录，若foo目录不存在，则建立一个：mkdir -p foo/test

rmdir

  • 作用：删除空目录
  • 语法：rmdir [-p] dir-name
  • 说明
    • -p选项：当子目录被删除会使父目录为空目录的话，则一并删除
  • 举例
    • 删除名为foo的空目录：rmdir foo
    • 删除foo目录中名为test的子目录，若test目录删除后foo目录变为空目录，则也被删除：rmdir -p foo/test
    • 删除名称以foo开始的空目录：rmdir foo*

rm

  • 作用：删除文件或目录
  • 语法：rm [-rf] name
  • 说明
    • -r选项：将目录及目录中所有文件（目录）逐一删除，即递归删除
    • -f选项：无需确认，直接删除
  • 举例
    • 删除名为foo的目录和目录中所有文件，删除前需确认：rm -r foo/
    • 无需确认，直接删除名为foo的目录和目录中所有文件：rm -rf foo/
    • 无需确认，直接删除hello.txt文件：rm -f hello.txt

拷贝移动命令

cp

  • 作用：用于复制文件或目录
  • 语法：cp [-r] source dest
  • 说明
    • -r选项：如果复制的是目录则需使用此选项，此时将复制该目录下所有的子目录和文件
  • 举例
    • 将hello.txt复制到foo目录中：cp hello.txt foo/
    • 将hello.txt复制到当前目录，并改名为hi.txt：cp hello.txt ./hi.txt
    • 将foo目录和目录下所有文件复制到bar目录下：cp -r foo/ ./bar/
    • 将foo目录下所有文件复制到bar目录下：cp -r foo/* ./bar/

mv

  • 作用：为文件或目录改名，或将文件或目录移动到其他位置
  • 语法：mv source dest
  • 举例
    • 将hello.txt改名为hi.txt：mv hello.txt hi.txt
    • 将文件hi.txt移动到bar目录中：mv hi.txt bar/
    • 将文件hi.txt移动到bar目录中，并改名为hello.txt：mv hi.txt bar/hello.txt
    • 如果bar目录不存在，将foo目录改名为bar；如果bar目录存在，将foo目录移动到bar目录中：mv foo/ bar/

打包压缩命令tar

  • 作用：对文件进行打包、解包、压缩、解压
  • 语法：tar [-zcxvf] file-name [files]
    • 包文件后缀为.tar表示仅完成了打包，并没有压缩
    • 包文件后缀为.tar.gz表示打包的同时还进行了压缩
  • 说明
    • -z选项：z代表gzip，通过gzip命令处理文件，gzip可以对文件压缩或解压
    • -c选项：c代表create，即创建新的包文件
    • -x选项：x代表extract，实现从包文件中还原文件
    • -v选项：v代表verbose，显示命令的执行过程
    • -f选项：f代表file，用于指定包文件的名称
  • 举例
    • 打包
      • 将当前目录下所有文件打包，打包后的文件名为hello.tar：tar -cvf hello.tar ./*
      • 将当前目录下所有文件打包并压缩，打包后的文件名为hello.tar.gz：tar -zcvf hello.tar.gz ./*
    • 解包
      • 将hello.tar文件进行解包，并将解包后的文件放在当前目录：tar -xvf hello.tar
      • 将hello.tar.gz文件进行解压，并将解压后的文件放在当前目录：tar -zxvf hello.tar.gz
      • 将hello.tar.gz文件进行解压，并将解压后的文件放在/usr/local目录：tar -zxvf hello.tar.gz -C /usr/local

文本编辑命令

vi

  • 作用：vi命令是Linux系统提供的一个文本编辑工具，可以对文件内容进行编辑，类似Windows中的记事本
  • 语法：vi file-name
  • 说明
    • vim是从vi发展来的一个功能更加强大的文本编辑工具，在编辑文件时可以对文本内容进行着色，方便对文件进行编辑处理，所以实际工作中vim更加常用

vim

  • 作用：对文件内容进行编辑，其实就是一个文本编辑器
  • 语法：vim file-name
  • 说明
    • 在使用vim命令编辑文件时，如果指定的文件存在，直接打开文件，否则新建文件
    • vim在进行文本编辑时分为命令模式、插入模式、底行模式三种模式，它们可以相互切换，所以使用vim时需注意当前所处的是何种模式
    • 命令模式
      • 可以查看文件内容、移动光标（上下左右箭头、gg、G）
      • 通过vim命令打开文件时默认进入的模式
      • 另两种模式只能从命令模式进入
    • 插入模式
      • 可以对文件内容进行编辑，下方出现【insert】字样
      • 在命令模式下按[i, a, o]的任意一个，可以进入插入模式
      • 按ESC键回到命令模式
    • 底行模式
      • 通过命令对文件内容进行查找、显示行号、退出等操作
      • 在命令模式下按[:, /]的任意一个，可以进入底行模式
      • 通过/进入底行模式后，可以对文件内容进行查找
      • 通过:方式进入底行模式后，可以输入wq（保存并退出）、q!（不保存退出）、set nu（显示行号）

查找命令

find

  • 作用：在指定目录下查找文件
  • 语法：find dir-name -option file-name
  • 举例
    • 在当前目录及其子目录下查找.java结尾的文件：find . -name "*.java"
    • 在/foo目录及其子目录下查找.java结尾的文件：find /foo -name "*.java"

grep

  • 作用：从指定文件中查找指定的文本内容
  • 语法：grep word file-name
  • 举例
    • 查找HelloWorld.java文件中出现的Hello字符串的位置：grep Hello HelloWorld.java
    • 查找当前目录中所有.java结尾的文件中包含hello字符串的位置：grep hello *.java

软件安装

软件安装方式

  • 二进制发布包安装：软件已经针对具体平台编译打包发布，只需解压、修改配置即可
  • rpm安装：软件已按照redhat的包管理规范进行打包，使用rpm命令进行安装，不能自行解决库依赖问题
  • yum安装：一种在线软件安装方式，本质上还是rpm安装，自动下载安装包并安装，安装过程中自动解决库依赖问题
  • 源码编译安装：软件以源码工程的形式发布，需要自己编译打包

安装Tomcat

操作步骤

  1. 使用FinelShell自带的上传工具将Tomcat的二进制发布包上传到Linux
  2. 解压安装包，命令为tar -zxvf file-name -C location
  3. 进入Tomcat的bin目录启动服务，命令为sh startup.sh或./startup.sh

验证Tomcat是否启动成功

  • 查看启动日志
    • more /usr/local/file-name/logs/catalina.out
    • tail -50 /usr/local/file-name/logs/catalina.out
  • 查看进程：ps -ef|grep tomcat
  • 注意
    • ps命令是Linux下非常强大的进程查看命令，通过ps -ef可以查看当前运行的所有进程的详细信息
    • Linux管道符|，可以将前一个命令的结果输出给后一个命令作为输入
    • 使用ps命令查看进程时，经常配合管道符和查找命令grep一起使用，来查看特定进程

防火墙操作

  • 查看防火墙状态：systemctl status firewalld 或firewall-cmd --state
  • 暂时关闭防火墙：systemctl stop firewalld
  • 永久关闭防火墙：systemctl disable firewalld
  • 开启防火墙：systemctl start firewalld
  • 开放指定端口：firewall-cmd --zone=public --add-port=8080/tcp --permanent
  • 关闭指定端口：firewall-cmd --zone=public --remove-port=8080/tcp --permanent
  • 立即生效：firewall-cmd --reload
  • 查看开放的端口：firewall-cmd --zone=public --list-ports
  • 注意
    • systemctl是管理Linux中服务的命令，可以对服务进行启动、停止、重启、查看状态等操作
    • firewall-cmd是Linux中专门用于控制防火墙的命令
    • 为保证系统安全，服务器的防火墙不建议关闭

停止Tomcat服务

  • 运行Tomcat的bin目录中提供的停止服务的脚本文件，命令为sh shutdown.sh或者./shutdown.sh
  • 结束Tomcat进程
    • 使用命令ps -ef|grep tomcat查看Tomcat进程，获得进程id
    • 执行命令kill -9 id，结束进程
  • 注意
    • kill命令是Linux提供的用于结束进程的命令，-9代表强制结束

nohup

  • 全称：no hang up（不挂起）
  • 作用：用于不挂断地运行指定命令，退出终端不会影响程序的运行
  • 语法格式：nohup Command [Arg ...] [&]
  • 说明
    • Command：要执行的命令
    • Arg：一些参数，可以指定输出文件
    • &：让文件在后台运行
  • 举例
    • 后台运行java -jar命令，并将日志输出到hello.log文件：nohup java -jar boot工程.jar &> hello.log &

通过Shell脚本自动部署项目

操作步骤

  1. 在Linux中安装Git

  2. 在Linux中安装Maven

  3. 编写Shell脚本（拉取代码、编译、打包、启动）

  4. 为用户授予执行Shell脚本的权限

    • chmod(change mode)命令是控制用户对文件的权限的命令

    • Linux的权限分为读®、写(w)、执行(x)三种权限

    • Linux的文件调用权限分为三级：文件所有者(Owner)、用户组(Group)、其他用户(Other Users)

    • 只有文件的所有者和超级用户可以修改文件或目录的权限

    • 要执行Shell脚本，需要有对此脚本文件的执行权限，如果没有则不能执行

    • chmod命令可以用八进制数来指定权限

      # 权限 rwx
      7 读+写+执行 rwx
      6 读+写 rw-
      5 读+执行 r-x
      4 只读 r–
      3 写+执行 -wx
      2 只写 -w-
      1 只执行 –x
      0 无 —
      • 举例
        • chmod 777 bootStart.sh：为所有用户授予读、写、执行权限
        • chmod 755 bootStart.sh：为文件拥有者授予读、写、执行权限，同组用户和其他用户授予读、执行权限
        • chmod 210 bootStart.sh为文件拥有者授予写权限、同组用户授予执行权限，其他用户没有权限
      • 注意：三位数字分别代表不同用户的权限
        • 第一位表示文件拥有者的权限
        • 第二位表示同组用户的权限
        • 第三位表示其他用户的权限
