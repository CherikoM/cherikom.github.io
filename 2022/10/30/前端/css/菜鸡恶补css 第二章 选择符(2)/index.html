伪类选择符

伪类选择符可以为文档中不一定真实存在的结构指定样式，或者为某些元素（甚至文档本身）的特定状态赋予“幽灵类”。

拼接伪类

  • css允许把伪类拼接（串联）在一起。
  • 拼接伪类的效果与伪类顺序无关，a:link:hover和a:hover:link效果上没有区别。
  • 如果拼接互斥的伪类会导致选择符没有意义，匹配不了任何东西。

结构伪类

伪类多数是结构上的，即指代文档中的标记结构，还有一些则用于选定特定的元素类型。

所有伪类都以冒号:词语为格式，可以出现在选择符的任意位置。

伪类始终指代所依附的元素（不要混淆:first-child等伪类的用法）。

选择根元素

:root伪类选择文档的根元素，在HTML中，根元素始终是html。

这个选择符的真正用途体现在XML中，在不同的XML语言中，根元素有所不同。

在HTML文档中可以直接选择html元素而不用:root伪类，但这两种选择符的特指度不同。

选择空元素

:empty伪类选择没有任何子代（包括文本节点）的元素。

标签间的空格和换行符也会被算作文本节点，但注释不算。

:empty能匹配到img、input和没有输入内容的textarea。

img和img:empty在选择角度是相等的，但具有不同的特指度。

选择唯一的子代

:only-child伪类可以选择某一元素的唯一子元素。

text text text text text

text text text text text

text text text text text

如果想特指唯一的某类子元素，可以使用:only-of-type，这里的“某类”一定指的是元素。

text text text text text

text text text text text

与:only-of-type类似，也可以使用:first-of-type和:last-of-type限制元素种类。

:nth-child()也有对应的:nth-last-child()，以倒数顺序选择。

同样，也有对应的:nth-of-type()和:nth-last-of-type()。

动态伪类

除了结构伪类外，还有一些与结构有关的伪类，但它们在页面渲染后会根据页面的变化而变化。

为了把这些伪类定义的样式应用到文档中的某处，除了文档的结构，还要考虑其他因素，而且这些样式是在无法提前预知的条件满足时运用的。

超链接伪类

css2.1定义的两个只能在超链接上使用的伪类。

  • :link指代用作超链接的锚记（且具有href属性），而且指向未访问的地址。
  • :visited指代指向已访问地址的超链接。

在HTML中，这两个伪类应用在具有href属性的a元素上；在XML中，这两个伪类应用在链接到其他资源的元素上。

为了提升可用性和可访问性，已访问和未访问链接间要能轻易区分。

只能把颜色相关的属性应用到已访问链接上，除此之外的属性将会被忽略。

:link实际上能装饰所有超链接，不仅是未访问过的超链接。

如果使用DOM查询已访问链接的样式，返回的值会和未访问时一样。

这一切都是出于信息安全考虑。

用户操作伪类

用户操作伪类可以根据用户的操作改变文档的外观。

  • :focus指代当前获得焦点的元素
  • :hover指代鼠标放置其上的元素
  • :active指代由用户输入激活的元素

可以处于:active状态的元素有链接、菜单项目、可以设定tabindex的元素，这些元素加上其他所有交互元素（例如表单控件和可编辑内容的元素）还可与获得焦点。

这些伪类最常（但不仅限）用于超链接，推荐使用顺序：

虽然可以使用:focus以任何方式装饰元素，但千万不要删除获得焦点的元素的全部样式，区分当前获得焦点的元素对可访问性十分重要，尤其是使用键盘在网站或应用中导航时。

UI动态伪类

与动态伪类紧密相关的是用户界面状态伪类，会根据用户界面元素的当前状态应用样式。

  • :enabled指代启用的用户界面元素，即接受输入的元素
  • :disabled指代禁用的用户界面元素，即不接受输入的元素
  • :checked指代由用户或文档默认选中的单选按钮或复选框
  • :indeterminate指代既未选中也没有未选中的单选按钮或复选框
    • 这个状态只能由DOM脚本设定，不能由用户设定
  • :default指代默认选中的单选按钮、复选框或选项
  • :valid指代满足所有数据有效性语义的输入框
  • :invalid指代不满足数据有效性语义的输入框
  • :in-range指代输入的值在最小值和最大值之间的输入框
  • :out-of-range指代输入的值不在最小值和最大值之间的输入框
  • :required指代必须输入值的输入框
  • :optional指代无需输入一定值的输入框
  • :read-write指代可由用户编辑的输入框
  • :read-only指代不能被用户编辑的输入框

虽然UI元素的状态能被用户操作海边，但是UI状态伪类不是单纯动态的，因为它们还受文档结构或DOM脚本的影响。

启用和禁用的UI元素

HTML5可以把一个（或一组）用户界面元素标记为禁用的，禁用元素也能显示出来，但是无法选择、激活或与用户交互，这类元素可以用:disabled选中，而未禁用的元素可以用:enabled选中。

选择状态

某些UI元素可以选中或者不选，例如复选框和单选框，使用:checked可以匹配它们。

没有:unchecked伪类，为了实现“选中所有没被选择的元素”，可以使用:not()

可选元素默认是未被选中的，但文档编写人员可以通过元素标记中的checked属性切换状态，也可以使用脚本改变元素的选择状态。

除了选中和未选中外，还有一个不确定状态，这个状态只能由脚本或用户代理实现，在标记中做不到这一点。

:indeterminate存在的目的是从视觉上提示用户需要选中（或不选）某个元素，但这只是视觉上的效果，对元素底层状态没有影响，元素的底层状态只能是选中或未选中。

默认选项伪类

:default伪类匹配一组相似元素中取默认值的元素，常用于上下文菜单选项、按钮和选择列表。

如果有几个同名的单选按钮，最初被选中的那个匹配:default。

页面加载时选中的复选框匹配:default。

select元素中最初选中的一个或多个option匹配:default。

:default伪类还能匹配按钮和菜单选项。

可选性伪类

:required匹配必填的表单控件，:optional匹配没有required属性或required属性为false的控件。

除了伪类，匹配必填表单控件也可以使用属性选择符。

除了表单输入框外，其他元素既不能是必填的，也不能是可选的

有效性伪类

当用户输入的值满足所有数据验证条件时可以匹配:valid，否则匹配:invalid。

它们只适用于能检查数据有效性的元素

这两个伪类起不起作用取决于用户代理会不会向样式系统报告验证状态，因此某些情况下的效果可能会和预期不符。

范围伪类

在HTML5中可以用min和max属性界定用户输入的范围，拥有范围限制的元素，输入值在范围内匹配:in-range，否则匹配:out-of-range。

如果输入值不匹配步进值（step属性）而无效，但仍然在min和max范围内，那么所在的元素将匹配:invalid和:in-range，也就是说，在范围内的值也可能是无效的。

可变性伪类

:read-write表示输入框可由用户编辑，:read-only表示输入框不能被用户编辑。

:target伪类

url中有个片段标识符（#后面的内容，可以定位到拥有该id的元素），它所指向的文档片段（在css中）被称为目标，url片段标识符指向的目标元素可以使用:target伪类特别装饰。

:target定义的样式在以下两种情况不会被使用：

  • 页面的url中没有片段标识符
  • 页面的url中有片段标识符，但是文档中没有与之匹配的元素

:lang伪类

如果想根据文本使用的语言选择元素，可以使用:lang伪类。

在匹配方式上，:lang()与|=属性选择符类似。

伪类选择符与属性选择符间的主要区别是语言信息有多个来源，对属性选择符而言，元素自身必须有lang属性才能匹配，而:lang伪类能匹配设定了语言的元素的后代。

在HTML中，语言可以通过lang属性、meta元素、协议等判断，XML中使用xml:lang属性。

:lang伪类可以使用各种信息，而|=属性选择符只能用于含lang属性的元素，因此前者比后者更可靠。多是装饰特定语言的理想之选。

否定伪类

前文中所有的选择符都是肯定选择符，它们用于指定应该选择的东西，排除不匹配（即不选择）的东西。

如果想反过来选择不满足条件的元素，可以使用selectors level 3引入的否定伪类:not

:not依附在元素上，括号内是简单的选择符，根据W3C的定义，简单的选择符指：一个类型选择符、通用选择符、类选择符、ID选择符或伪类。

:not只能使用一种选择符，不能使用群组选择符，也不能使用连接符，更不能使用后代选择符。

否定伪类不能嵌套，不能引用伪元素，可以串联使用。

伪元素选择符

伪元素类似伪类，为了实现特定的效果，它在文档中插入虚构的元素。

css2定义了四个基本的伪元素，之后的规范又定义了其他伪元素。

所有伪元素只能出现在选择符的最后，出现在中间是无效的。

伪元素开头使用两个冒号::，为了向后兼容，浏览器接受使用一个冒号的伪元素，但不确保什么时候就不支持兼容了，所以应该使用正确的冒号个数。

装饰首字母

::first-letter用于装饰任何非行内元素的首字母单元，或开头的标点符号和首字母单元（如果文本以标点符号开头）

abcdefghijklmnopqrstuvwxyz

::first-letter经常用于实现排版效果中的“首字母大写’或”首字母下沉“

这个规则其实就相当于让用户代理装饰每个p元素中包围首字母的一个虚构元素。

装饰首行

::first-line用于装饰元素的首行文本。

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Fusce dignissim aliquam ligula, at dignissim urna. Vestibulum at ultrices odio. Quisque sed egestas orci. Vivamus eget pretium nunc, et laoreet lectus. Maecenas id libero eu nulla pharetra volutpat. Morbi vitae quam eget ex maximus semper. Mauris ac bibendum ipsum, non mattis lacus. Morbi ornare, velit eu bibendum aliquam, sapien mauris volutpat justo, ac pharetra nunc lorem id massa.

如果用户增大或减小字号、拉宽或缩窄浏览器窗口，导致文本的宽度有变，首行中的词数随之增多或减少，浏览器能自动调整。
