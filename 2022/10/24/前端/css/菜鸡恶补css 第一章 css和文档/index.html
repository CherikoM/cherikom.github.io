元素

置换元素和非置换元素

置换元素

置换元素指用来置换元素内容的部分不由文档内容直接表示。例如：

  • img，没有在文档中的内容，而是显示文档之外，src属性指向的图像文件。
  • input，根据类型的不同，会被替换成单选按钮、复选框、文本输入框等。

非置换元素

非置换元素的内容由用户代理在元素自身生成的框里显示。

在文档中的元素内写了什么内容，框里就显示什么内容。大部分HTML元素都是非置换元素

元素的显示方式

在HTML中，块级元素不能出现在行内元素内，但css不限制它们的显示方式，相互之间可以嵌套。

用css改变元素显示方式对XML文档的意义更大。

块级元素

块级元素的块独占一行，两旁没有其他元素。置换元素可以是块级元素，但往往不是

行内元素

行内元素在一行文本内生成元素框，不打断所在的行。

把css应用到HTML上

link标签

外部样式表包含一系列css样式声明，存储在单独的文件内。

使用外部样式有利于更新和维护网站。

如果外部样式表中有css以外的语法，如文档标记，可能导致部分或全部样式失效。

外部样式表的拓展名不是必须的，但如果拓展名不是css，即使link的属性type=‘text/css’，有些旧浏览器也不会将其识别为样式表。

使用link标签的href属性引入外部样式表供文档使用。

为了正确加载外部样式表，必须将link标签放在head元素内。

一个文档可以关联多个样式表，合并样式规则后将一并应用到文档上。

属性

  • rel：relationship，当前文档与加载文件之间的关系，引入外部样式表时设为stylesheet。
  • type：加载的数据类型，引入css外部样式表时设为text/css。
  • href：加载文件的URL。
  • media：指定应用加载文件的媒体类型，多个媒体描述符以逗号分开。

候选样式表

将属性rel='alternate stylesheet’后，仅当用户自己选择，文档才会使用候选样式表渲染，这样的样式表被称为候选样式表，而属性rel='stylesheet’将会成为首选样式表。

使用title属性可以将外部样式表命名，可供用户选择样式，不指定title属性则为永久样式表，始终用于显示文档（css早期的使用方式）。

style元素

style元素使得编写人员可以直接将样式表直接写在文档内，这样的样式表被称为文档样式表或嵌入式样式表。

style元素始终需要设定type属性，且值必须为’text/css’。

style元素的media属性功能与link元素一致。

也可以使用@import在style元素内引入外部样式表。

@import指令

与link元素相似，可以使用@import在样式表（不仅限于嵌入式样式表）内加载外部样式表。

@import指令必须位于样式表的顶部，否则不会起作用。

一个文档中可以有多个@import，它们所导入的每个样式表都会被使用。@import也可指定应用媒体。

style的值只能是一系列样式声明，而不能包含整个样式表，因此不能在行内样式中使用@import，也不能有完整的规则。

通常不建议也没必要使用行内样式，它抹去了css的优点，增加了样式与结构间的耦合度。

规则的结构

  • 一个样式表由一系列规则组成。
  • 一个规则由选择符和声明块两部分组成。
  • 一个声明块由一个或多个声明组成。
  • 一个声明由一个属性和对应的值组成。

选择符定义文档中受该样式影响的元素，声明块定义样式呈现什么效果。

厂商前缀

有些属性名前面会有个标注，被称为厂商前缀。

浏览器厂商通过它标注实验性或专属（或二者兼具）的属性、值或其他内容。

前缀 厂商
-epub- 国际数字出版论坛指定的ePub格式
-moz- 基于Mozilla的浏览器，如Firefox
-ms- 微软IE
-o- 基于Opera的浏览器
-webkit- 基于WebKit的浏览器，如Safari和Chrome

处理空格

css对规则之间，规则之内的空格基本都没什么严格要求，有部分情况会要求至少留一个空格。

一般来说，css处理空格的方式和HTML差不多，解析时，连续的空格会合并成一个空格。

css注释

css没有单行注释，只有多行注释，且不能嵌套。

在css解析器眼里，css注释相当于根本没出现过，所以不会被解析器算作空格。因此，注释可以放在规则，甚至声明内部。

媒体查询

通过媒体查询，浏览器在不同媒体环境可以使用不同的指定样式表。

用处

  • link元素的media属性
  • style元素的media属性
  • @import声明的媒体描述符部分
  • @media声明的媒体描述符部分

简单的媒体查询

一个样式表中可以有任意多个@media块，而且每一个都有自己的一套媒体描述符。

该例在投影媒体中，body会有一个黄色背景。

媒体类型

媒体类型是媒体查询最基本的形式，由css2引入，能指明不同媒体的标注：

  • all：所有媒体设备
  • aural：语音和音频合成去等听觉设备
  • braille：盲人用点字法触觉回馈设备
  • embossed：盲人用点字法打印机
  • handheld：小型手持设备，如手机、平板电脑
  • print：打印机
  • projection：投影设备
  • screen：电脑显示器等屏幕媒体
  • tty：使用固定密度字母栅格的媒体，如打字机
  • tv：电视

使用多个媒体类型时要用逗号罗列。

媒体描述符

媒体特性描述符放在圆括号中，格式类似css的键值对，但可以不指定值。

一个媒体描述符包含一个媒体类型和一或多个由媒体特性描述符组成的媒体特性列表。

当媒体描述符不含媒体类型时，默认应用到所有媒体上。

媒体查询关键字

  • and：连接的两个或多个媒体特性必须同时满足条件，整个查询结果才会是真值。
  • not：对整个查询结果取反，只能用于媒体查询的开头。
  • 媒体设备不支持or关键字，但分隔描述符用的逗号,相当于or。
  • only：在不支持媒体查询的旧浏览器中隐藏样式表，在支持媒体查询的浏览器中会被忽略，只能用于媒体查询的开头。

特性查询

特性查询能根据用户代理是否支持特定的css属性及值，决定对一段样式的应用与否。
