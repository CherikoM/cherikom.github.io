样式的基本规则

元素选择符

元素选择符通常是HTML元素，在XML文档中也可能是任意字符串，即，文档中的元素是最基本的选择符。

元素选择符可以将声明块里的样式应用到所有对应的元素上。

声明和关键字

不是每一个属性都能接受多个关键字。

通用选择符

css2引入通用选择符*，可以匹配所有元素，相当于列出文档所有元素的群组选择符。

通用选择符没有特指度，而且由于目标是一切元素，使用它可能会出现意想不到的后果。

一个规则中最后一个声明的结尾可以不带分号，但一般建议带上。如果使用类如Scss这样的预处理器，语法可能会强制要求结尾加上分号。

在旧浏览器中使用新元素

HTML会在更新的过程中添加新的元素，而有些浏览器因为太旧，可能无法识别新元素。

例如IE9以下版本中无法选择不支持的元素，解决方法是创建新元素，让浏览器知道元素的存在。

类选择符和ID选择符

除了直接使用文档中的元素外，还可以使用类选择符和ID选择符，以一种独立于元素的方式赋予样式。

这两种选择符可以独立使用，也可以结合元素选择符使用。

但它们要求文档的标记有一定的结构，因此要提前构思，做好规划。

类选择符

应用样式而不关心所涉及的元素，最常使用类选择符。

使用之前，要在文档中特定标签内设定class属性。

在class属性值前加上点号.，即为一个类选择器。

样式将运用到所有包含warning类的元素（因为这里蕴含着通用选择符）。

多个类

class属性的值可以是多个词，词之间使用空格分隔，词的顺序不影响效果。

把两个类选择符串在一起，选择的是同时具有两个类（缺一不可）的元素，而且类选择符的顺序不影响效果。

ID选择符

ID选择符以井号#开头。

ID选择符引用的是id属性的值。

与类选择符一样，ID选择符隐式蕴含通用选择符*，且不关心涉及的元素。

与类选择符不同，ID选择符不能串在一起使用，因为ID属性的值不能是以空格分隔的列表。

在类选择符和ID选择符之间选择

在同一HTML文档中，类可以赋予人任意个元素，而ID是唯一的，一个ID只能出现一次。

浏览器不一定总会检查HTML中的ID是否唯一，但不唯一的ID对JavaScript编程不利。

ID选择符的权重比类选择符更高。

在HTML规范中，类选择符和ID选择符区分大小写。

HTML、SVG和MathML支持点号+类名的写法，但其无法保证一定能用于XML文档。

井号+ID的写法支持任何能确保文档内元素唯一性的文档语言，唯一性可以通过id属性确保，也可能通过其他任何方式。

属性选择符

无论是类选择符还是ID选择符，选择的其实都是属性的值。

在一些标记语言中可能不存在class和id属性，所以css2引入了属性选择符。

属性选择符根据属性及其值选择元素。

属性选择符分为四类：简单属性选择符、精准属性值选择符、部分匹配属性值选择符、起始属性选择符。

这种选择符在XML文档中特别有用，因为XML语言中的元素和属性都是根据目的而选用的。

简单属性选择符

简单属性选择符可以选择具有某个属性的元素，不关心属性的值是什么。

我是大哥

我是二哥

我是小弟

可以基于多个属性选择。

我有href和title
我有href
我有title

根据精准的属性值选择

精准属性值选择符缩小了简单属性选择符的范围，只选择属性为特定值的元素。

选择器内的属性值必须和元素内的属性值完全一致，即精确匹配字符串。

所以精准属性值选择符不能像类选择符那样使用，选择id属性时也和ID选择符有微妙的区别。

我会被选中

我不会被选中

与简单属性选择符相同，可以将多个属性和值选择符串在一起。

根据部分属性值选择

部分匹配属性值选择符会需要根据属性值的一部分选择元素，而不是完整的值。

形式 说明
[foo|=“bar”] 选择的元素有foo属性且属性值以bar-开头或就是bar
[foo~=“bar”] 选择的元素有foo属性且属性值为包含词语bar的一组词语
[foo*=“bar”] 选择的元素有foo属性且属性值包含子串bar
[foo^=“bar”] 选择的元素有foo属性且属性值以bar开头
[foo$=“bar”] 选择的元素有foo属性且属性值以bar结尾

Greetings!

这种匹配最常用于匹配语言名。

匹配词语

匹配子串

cloud factory

cotton candy

这种选择符做的是精准匹配，区分大小写和空格，HTML关键字除外。

匹配属性值开头/结尾的子串

soft cloudy pillow

cotton candy

匹配结尾子串的选择符常用于进行连接扩展名的匹配。

不区分大小写的标识符

css selectors level4为属性选择符引入了不区分大小写的选项，即在右方括号前加i。

所有属性选择符都可以加上该选项，但它只针对属性选择符中的值，不涉及属性的名称（仅针对句法区分大小写的语言）。

cloud factory

cloudy pillow

cotton CLOUD
