文档的树状结构不仅使人能根据元素的祖辈、属性、同胞元素等创建选择符选择元素，还是css继承的核心。

继承是指把一个元素的某些属性值传给其后代的机制。确定应该把哪些值应用到元素上时，用户代理不仅要考虑继承，还考虑声明的特指度和声明的来源，这个过程即层叠。

特指度

同一个元素可能会被两个或多个规则选择，而且每个规则的选择符不尽相同，如果它们声明的某样式起冲突，用户代理会计算每个规则中选择符的特指度，特指度最高的声明能胜出并将该样式应用在元素上。

所有样式冲突（包括特指度）都由层叠处理。

选择符的特指度由选择符本身的组成成分决定，一个特指度值由四部分构成（0,0,0,0）：

  • 选择符中的每个ID属性值加0,1,0,0。
  • 选择符中的每个类属性值、属性选择或伪类加0,0,1,0。
  • 选择符中的每个元素和伪元素加0,0,0,1。
  • 连接符和通用选择符不增加特指度。

特指度值是从左向右比较的，而且不存在进位。

声明和特指度

选择符的特指度确定之后，其值将赋予关联的每个声明。

为了计算特指度，用户代理必须把规则“打散”成单独的规则。

在任何情况下，用户代理都会确定哪些规则和元素匹配，然后找出所有相关声明，计算各自的特指度，判断哪些规则胜出，再将胜出的规则应用到元素上，得到装饰后的结果。

每个元素、选择符都要经历这一系列操作，用户代理能自动处理这一切。这个行为是层叠的重要部分。

通用选择符的特指度

通用选择符不增加特指度，即它的特指度为0,0,0,0，但这与没有特指度是不同的。

包含通用选择符的选择符，它的特指度不因拥有通用选择符而发生改变。

而连结符不同，它根本没有特指度，因此连结符对选择符的总特指度没有影响。

ID和属性选择符的特指度

ID选择符和选择id属性的选择符在特指度上是有区别的。

ID选择符贡献的特指度为0,1,0,0，属性选择符贡献的特指度为0,0,1,0。

行内样式的特指度

行内样式声明的特指度比其他声明都高，特指度为1,0,0,0。

重要性

重要声明非常重要，需要超过其他所有声明，这种声明要在声明末尾的分号前加一个!important。

如果想把两个声明都标记为重要的，每个声明中都要插入!important。

!important的位置必须正确，否则声明将失效。

带有!important的声明对特指度并非没有影响，但是会与不重要的声明分开处理。

所有带!important的声明会被放在一起，而特指度冲突就在这个范围内解决；同样，非重要的声明作为一个整体，其中的冲突用特指度解决。

重要声明和非重要声明冲突时，重要声明始终胜出。

继承

继承指某些样式不仅应用到所指的元素上，还应用到元素的后代上。

继承属性只沿文档树向下传播给后代元素，直至没有后代为止，绝不向上传播。

有一个例外是应用到body元素的背景样式会传给html元素，这一例外仅发生在为body定义了背景而没有为html定义的情况下。

为了避免意外的结果，有很多属性不会被继承，例如border属性、盒模型属性等。

其次继承的值没有特指度，连零也没有，因此通用选择符样式能胜过继承样式。（滥用通用选择符的潜在危险：终结继承）。

使用inherit关键字能强制元素使用继承样式。

层叠

css全称“层叠样式表”，它采用层叠机制把样式组合在一起，即结合继承和特指度的一些规则。

css的层叠规则：

  • 找到匹配特定元素的所有规则。
  • 按显示权重排序应用到特定元素上的所有声明，有!important的规则比没有的权重高。
  • 按来源排序应用到特定元素上的所有声明。
    • 声明有三个来源：创作人员、读者和用户代理。
    • 正常情况下：
      • 创作人员编写的样式击败读者提供的样式。
      • 读者样式中以!important标记的声明比其他样式权重高，包括创作人员编写的!important声明。
      • 创作人员和读者样式覆盖用户代理的默认样式。
  • 按特指度排序应用到特定元素上的所有声明，特指度高的声明具有较高的权重。
  • 按声明的前后位置排序应用到特定元素上的所有声明，样式表或文档书写靠后的声明权重较高，导入的样式表中的声明放在当前样式表所有声明的前面。

较新发布的某些css模块为第三条规则添加了更多的，例如动画和过渡，的来源。

按权重和来源排序

  1. 有!important标记的规则胜过没有的规则。
  2. 如果!important在行内样式中，行内样式胜出。
  3. 如果权重相同，一个在创作人员编写的样式表中，另一个在读者提供的样式表中，前者胜出。
  4. 同2的前提，但如果两个规则都用!important标记，后者胜出。
  5. 在所有来源中，用户代理的默认样式声明是影响力最低的。

综上，下面按权重从高到低列出：

  1. 读者提供的样式中以!important标记的声明
  2. 创作人员编写的样式中以!important标记的声明
  3. 创作人员编写的常规声明
  4. 读者提供的常规声明
  5. 用户代理的默认声明

创作人员只需考虑前四点，因为创作人员编写的任何样式都会覆盖用户代理样式。

按特指度排序

如果应用到一个元素上的声明有冲突，而且各声明的显式权重和来源相同，那么应该按特指度排序，特指度最高的声明胜出。

按前后位置排序

如果两个规则的显式权重、来源和特指度都相同：

  • 在样式表中的位置靠后的规则胜过靠前的规则。
  • 当前样式表的样式胜过引用样式。

就是因为前后位置有影响，所以编写链接样式时适合使用LVFHA顺序。

css之外的表现提示

文档除了css外还可能包含表现提示，例如font元素。

这种表现提示的特指度为0，而且被认为出现在创作人员编写的样式表的开头。

表现提示将被创作人员或读者提供的样式覆盖，但不会被用户代理的默认样式覆盖。

css3把css外部的表现提示视作用户代理默认样式表的一部分，而且假定出现在默认样式表最后。
