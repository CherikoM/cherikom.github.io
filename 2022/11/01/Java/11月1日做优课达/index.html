1.2 Maven入门（上）

知识点

Maven是一个项目管理和构建自动化工具，一般最关注它的项目构建功能。

Maven可以把工程打包成Java支持的格式（如jar），并支持部署到中央仓库里，以便使用者快捷地运用其他人写的代码，只要添加依赖就可以。

借助于Maven的中央仓库，可以把Java代码任意地共享给别人，这对团队协同开发而言至关重要。

项目目录

Maven使用惯例优于配置的原则，它要求在没有定制之前，所有项目都有如下的结构，其中${basedir}表示Java工程的根目录。

目录 目的
${basedir} 存放pom.xml和所有的子目录
${basedir}/pom.xml Maven的项目配置文件
${basedir}/src/main/java 项目的Java源代码
${basedir}/src/main/resource 项目的资源，比如property文件
${basedir}/src/test/java 项目的测试类，比如JUnit代码
${basedir}/src/test/resource 测试使用的资源
${basedir}/target/classes 编译后存放classes的位置
${basedir}/target 编译后存放jar包的位置

Maven命令

常用命令：

  • mvn clean compile：编译命令，Maven会自动扫描src/main/java下的代码并完成编译工作，执行完后会在根目录下生成target/classes目录。
  • mvn clean package：编译并打包命令，这个命令是compile和package的集合，先会执行compile命令，再执行jar打包命令，结果会把所有的java文件和资源打包成一个jar包。
  • mvn clean install：执行安装命令，这个命令是compile、package和install的集合，先会执行compile、jar打包，然后执行install命令安装到本地的Maven仓库目录里，这个目录是${user_home}/.m2，${user_home}即电脑登陆用户名的个人目录。
  • mvn compile exec:java -Dexec.mainClass=${main}：在compile执行完后，执行运行Java的命令，具体执行哪个Java由后面的参数决定。

1.2 Maven入门（中）

知识点

Maven的核心概念：POM、仓库、依赖管理、插件、生命周期。

这五个概念都会运用在Maven的配置文件中，它是一个强约定的XML格式文件，名为pom.xml。

POM（Project Object Model）

一个Java项目所有的配置都放置在POM文件中，大概有如下行为：

  • 定义项目的类型、名字
  • 管理依赖关系
  • 定制插件

包含如下内容：

  • Maven坐标
  • Maven工程属性
  • Maven依赖
  • Maven插件

Maven坐标

坐标是一种位置信息，Maven的坐标决定了这个Maven工程部署后存在Maven仓库的文件位置，所以它必须指定。

  • groupId：命名和Java的包比较一致，一般一个公司会设置自己的groupId以避免和其他公司重合，个人开发者也一样。
  • artifactId：类似文件名，在一个groupId内，它应该是唯一的。
  • packaging：工程打包的格式，默认为jar，可选jar、war、ear、pom四种。
  • version：版本号，遵循软件工程对版本号的约定。
    • SNAPSHOT：快照，代表当前开发还处于不稳定的阶段，随时可能再修改
    • RELEASE：稳定，发行版的版本，默认值。

Maven属性配置

Maven的属性配置用来做参数设置，在properties标签内。

  • java.version：代表设置一个公共参数，它的值是1.8。
  • maven.compile.source：指定Maven编译时源代码的JDK版本，使用了java.version的值。
  • project.build.sourceEncoding：工程代码源文件的文件编码格式。
  • maven.compile.target：按照这个值编译源代码。

1.4 Maven入门（下）

知识点

依赖管理

dependency用于指定当前工程依赖的其他代码库，Maven会自动管理jar依赖，只需要声明即可。

Maven会先去本地用户的.m2文件夹查找对应文件，找不到就去中央仓库下载。

一个pom.xml只能存在一个dependencies标签，可以内涵多个dependency，dependency标签内其实就是Maven坐标。

如果项目依赖的某个远程依赖依赖了另一个依赖，Maven工程会自动依赖全部相关的包。

插件体系

插件体系让Maven变得高度可定制，所以可以无缝支撑工程化能力，不同的插件有不同的作用。

插件也是存放在中央仓库的坐标，一切皆是jar包。

面向接口开发

在Spring中，如果想调用某接口就可以直接从上下文获取，而不必关心它的实现类以及如何实现，从而达到了真正的解耦合。

它完全屏蔽了接口的实现细节，让使用者可以专注于接口的定义和运用，对多人协同开发非常有用。
