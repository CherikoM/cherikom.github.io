2.1 Java注解

知识点

注解Annotation是Java推出的一种注释机制，和普通注释的显著区别是：注解可以在编译、运行阶段读取，而注释不可以。

注解给代码带来了很多的扩展空间，而且不会污染源代码，可以借助它实现一些增强功能。

Spring中重度使用了注解，通过运行阶段动态的获取注解，从而完成很多自定义的行为。

注解就是一个特殊的Java类，注解类里可以继续引用其他的注解类。

注解类的结构

① Target

java.lang.annotation.Target只有一个数组属性，用于设定该注解的目标范围，比如可以作用于类或方法等。

因为是数组，所以可以同时设定多个范围，具体可以作用的类型配置在java.lang.annotation.ElementType枚举类里。

例：

ElementType.TYPE

可以作用于类、接口类、枚举类上。

ElementType.FIELD

可以作用于类的属性上。

ElementType.METHOD

可以作用于类的方法上。

ElementType.PARAMETER

可以作用于类的参数。

② Retention

java.lang.annotation.Retention用于声明注解的生命周期，即在Java编译、运行的哪个环节有效。

它的值定义在java.lang.annotation.RetentionPolicy枚举类里，有三个值可以选择：

  • RetentionPolicy.SOURCE：纯注释作用。
  • RetentionPolicy.CLASS：在编译阶段有效。
  • RetentionPolicy.RUNTIME：在运行时有效。

③ Documented

java.lang.annotation.Documented用于将注解中的元素包含到JavaDoc文档中，注解类一般都会添加这个注解。

@interface声明当前的类是注解

类似普通类的属性，约定了属性的类型（基础类型）和属性名称（默认为value，引用时可省略），default代表默认值。

2.2 Spring Bean（上）

知识点

IoC（Inversion of Control，控制反转）是面向对象编程中的一种设计原则，可以减低代码间的耦合度，也是Spring框架最核心的组件，没有IoC容器就没有Spring框架。

Spring中主要通过依赖注入（DI，Dependency Injection）实现IoC

所有的Java对象都可以被Spring的IoC容器转变为Bean（Spring对象的一种称呼），构成应用程序主干和由Spring IoC容器管理的对象称为beans。

beans和它们之间的依赖关系反映在容器使用的配置元数据中，基本所有的Bean都是由接口+实现类完成的，用户若想获取Bean的实例可以直接从IoC容器获取，无需关心实现类。

Spring主要有两种配置元数据的方式，一种是基于XML，另一种是基于注解。主流方案是基于注解。

org.springframework.context.ApplicationContext接口类定义容器的对外服务，通过这个接口可以轻松从IoC容器中得到Bean对象。启动Java程序时必须先启动IoC容器。

运行这行代码启动IoC容器

AnnotationConfigApplicationContext这个类的构造函数有两种：

  • AnnotationConfigApplicationContext(String basePackages)根据包名实例化。
  • AnnotationConfigApplicationContext(Class clazz)根据自定义包扫描行为实例化。

Spring Bean的注解有以下几种：

  • org.springframework.stereotype.Component：@Component，通用的Bean注解，其他注解都是它的扩展。
  • org.springframework.stereotype.Service：@Service，代表Service Bean。
  • org.springframework.stereotype.Controller：@Controller，用于Web Bean
  • org.springframework.stereotype.Repository：@Repository，用于持久化相关Bean

这四个注解都可以被IoC容器加载，一般使用@Service，如果是Web服务器就使用@Controller。

Spring Bean（下）

知识点

启动后的第二步就是真正地完成依赖注入。

依赖注入也是一种编程思想，一种获取其他实例的规范。

如果某一实现类依赖的接口太多，那么就要大量编码，new很多服务实例，导致项目极易出错，勘误也比较困难。

Spring可以用@Service把服务实现，都标记成Spring Bean，然后在任何需要使用服务的地方用@Autowired注解标记，告诉Spring这里要注入实现类的实例。

项目启动过程中，Spring会自动实例化服务实现类，然后自动注入到变量中，不需要开发者再大量的new了。

@Auriwired完整的类路径是org.springframework.beans.factory.annotation.Autowired。

2.4 Spring Resource（上）

知识点

Spring Framework作为完整的Java企业级解决方案，也拥有自己的文件处理方案：Spring Resource。

当文件位于src/main/resources目录下时，打包后资源路径可能出问题（Maven工程编译后会去掉src/main/java和src/main/resources目录，jar文件是从包路径开始的）。

classpath

在Java内部，一般把文件路径称为classpath，所以读取内部的文件就是从classpath内读取。

classpath指定的文件不能解析成File对象，但是可以解析成InputStream，借助Java IO可以读取出来。

classpath类似虚拟目录，根目录从/开始，代表src/main/java或src/main/resources目录。

3.1 Spring Boot介绍

知识点

Spring Boot是Spring团队打造的面向微服务的框架（严格来说它不能算一个框架，它更像是一个方案）。

它极大地降低了Java Web工程的创建、运行和部署的难度。

Spring Boot的核心还是Spring，只是多了一些工程化的方案，例如：

  • Java Web容器的嵌入集成（默认继承了Tomcat，所以就不用再额外部署Tomcat了）。
  • 自定义工程打包格式，能直接把一个Java Web工程转化为普通的Java工程，启动一个main方法就可以启动Spring工程，极大地降低了开发难度。
  • 默认集成了大量第三方框架和服务，如数据库连接、NoSQL、安全等，不用担心复杂的Maven依赖，开箱即用。
  • 提供了标准的属性配置文件，支持应用的参数动态配置，让代码运行更加灵活强大。

在官网Spring Initializr可以创建一个Spring Boot工程。

3.2 Spring Controller入门

知识点

Java Web的规范是Servlet技术，所以所有的Java Web都实现了Servlet API的定义。

随着响应式编程技术的普及，Servlet也不再是唯一的标准了。

过程：一个网页请求进入到服务器后，首先进入的是Java Web服务器，然后进入到Spring Boot应用，最后匹配到某一个Spring Controller，然后路由到具体某一个Bean的方法，执行完后返回结果，输出到客户端。

只要掌握Spring Controller就可以自己提供一个Web服务了。

Spring Controller技术的三个核心点：

  • Bean的配置：Controller注解运用
  • 网络资源的加载：加载网页
  • 网址路由的配置：RequestMapping注解运用

一般会把controller放在control子包里。

Controller注解

Spring Controller本身也是一个Spring Bean，只是它多提供了一个Web能力，只需在类上提供一个@Controller注解即可

加载网页

在Spring Boot应用中，一般把网页存放在src/main/resources/static目录下。

controller会自动加载static下的html内容。

RequestMapping注解

只要在提供Web访问的方法上添加@RequestMapping注解就可以完成路由配置。
