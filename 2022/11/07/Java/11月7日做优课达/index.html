6.2 Spring Boot Logger运用

知识点

介绍日志系统

在Spring这种比较复杂的系统中，System.out.println()打印的内容会输出到什么地方是不确定的，所以在企业级的项目中，都会用日志系统来记录信息。

日志系统的两大优势：

  • 轻松地控制日志是否输出
    • 例如淘宝这类特大型网站，在开发阶段需要打印出调试信息，但发布到正式环境就不能打印了，因为每天几百亿的访问量，大量调试信息会导致磁盘爆满，这时就要控制日志的输出，而不是修改所有的代码。
  • 灵活地配置日志的细节
    • 例如输出格式，通常在日志输出时，需要自动附带输出日志发生的时间、打印日志的类名等信息，这样能很方便的观察日志分析问题。

常用的日志级别（优先级）：

优先级 级别 含义和作用
最高 ERROR 错误信息日志
高 WARN 暂时不出错但高风险的警告信息日志
中 INFO 一般的提示语、普通数据等不紧要的信息日志
低 DEBUG 进开发阶段需要关注的调试日志信息

logging.level.root=error意味着不输出更低优先级的WARN、INFO、DEBUG日志，只输出ERROR日志

logging.level.root=warn意味着不输出更低优先级的INFO、DEBUG日志，只输出WARN和更高优先级的ERROR日志。

以此类推。

在开发阶段配置为DEBUG，在项目发布时调整为INFO或更高级别，即可做到不改代码而控制只输出关心的日志。

6.3 Spring Boot Properties

知识点

Spring Boot框架已经免除了大部分手动配置，但对于一些特定情况，可能还是需要配置Spring Boot。

框架提供了application.properties文件，以对默认的配置进行修改。

文件在项目的src/main/resources/目录，框架会自动加载并解析这个配置文件

配置格式

每一行是一条配置项：配置项名称=配置项值

  • 等号两边不加空格，要写得紧凑
  • 配置项名称能准确表达作用、含义，以点.分割单词
  • 相同前缀的配置项写在一起
  • 不同前缀的配置项之间空一行

配置的意义

配置的主要作用是把可变的内容从代码中剥离出来，做到在不修改代码的情况下，方便地修改这些可变的或常变的内容，这个过程称之为避免硬编码，做到解耦。

通常跟项目运行相关的上下文环境，比如端口号、路径等可能变化信息，是可变或常变的内容。
