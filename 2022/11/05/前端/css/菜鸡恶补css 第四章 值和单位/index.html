关键字、字符串和其他文本值

样式表中的一切都是文本，但有些类型的值是字符串本身，而不是数字或颜色等。

关键字

接受关键字的属性，所取的关键字必须在哪个属性允许使用的关键字范围之内。

两个使用相同关键字的不同属性可能存在不同的行为。

全局关键字

css3中定义的全局关键字inherit、initial和unset，在规范中的每个属性都能使用。

inherit

inherit关键字把元素某个属性的值设为与父元素同一属性的值相同，即，这个关键字进行强制继承，即使在通常情况下不会继承（如border）。

One Two Three

按照用户代理的设置，链接一般会被显示成蓝色，使用inherit即可无视默认样式，强制让链接继承样式。

该关键字兼容性较差，且截至到目前无法在safari和Firefox使用。

initial

initial关键字把属性的值设为预定义的初始值/默认值，相当于“重设值”。

例如font-weight的默认值为normal，因此font-weight: initial;的作用与font-weight: normal;一样。

再如color，它没有自己的默认值，默认值取决于浏览器等的偏好设置，部分用户可能会修改偏好，所以color: initial;的作用为将文字颜色根据用户的设置显示。

unset

unset关键字是inherit和initial的通用替身。

对继承的属性来说，unset的作用与inherit一样；对不继承的属性来说，unset的作用与initial一样。

all

上述三个全局关键字在所有属性中都可以使用，有个特殊的属性只接受这几个关键字：all。

all表示出direction和unicode-bidi之外的所有属性，因此，如果为一个元素声明all: inherit;，意为除direction和unicode-bidi之外的所有属性都从父元素上继承值。

字符串

字符串值放在单引号或双引号内的任意字符序列，定义时可取的值以<string>表示。

前后引号种类要一致，引号内可以使用不同种类引号，或用反斜杠\转义引号。

反斜杠\也可以转义换行符。

URL

取值句法用<url>表示，是引用url的一般格式，使用绝对url和相对url都可以：

在css中，相对url应该相对于所在的样式表，而非使用样式表的文档。

图像

取值句法用<image>表示，表示为下面三种值的一个：

  • <url>：指向外部资源的url标识符，这里指图像的url。
  • <image-set>：一系列图像，根据值中的条件选择。
  • <gradient>：线性渐变或径向渐变图像。

标识符

有些属性接受用户定义的某种标识符值，最常见的有生成的列表序号，取值句法用<identifier>表示。

标识符本身是词，而且区分大小写，且不能与属性预定义的关键字重名。

数字和百分数

整数

由一到多个整数组成，可以是负数，取值句法用<integer>表示。

取值范围外的整数值会被默认是无效的，会导致整个声明被忽略。如果属性未明确数值范围，用户代理必须接受±1073741824(±2302^{30}230)

然而根据某些属性的定义，取值范围外的整数会被设为与所用值最接近的数，这种行为被称为clamping。

数字

数字值即实数，取值句法用<number>表示，是<integer>的超集，有些属性可能只接受整数取值，但有些属性也能接受小数取值。

百分数

百分数是<number>后跟一个百分号的值，取值句法用<percentage>表示。

百分数值始终相对于另一个值，另一个值可能是继承自父元素或祖辈元素的值，或同一元素另一属性的值，等等。

接受百分数值的属性通常会定义取值范围，以及百分比相对什么计算。

弹性值

弹性值就是<nunber>后跟fr，因此单位弹性值为1fr，由栅格布局引用，用于把布局中不受限制的空间分成几部分。

距离

很多css属性都需要衡量长度以正确显示页面中的各个元素，所以css有很多度量长度的方式。

长度值由正数或负数后跟一个长度单位单位表示，不过也有些属性只接收正数。

当长度为0时可以不用添加单位。

长度单位分为绝对长度单位和相对长度单位两种。

绝对长度单位

绝对长度单位有七个，但Web设计中很少使用：

  • 英寸(in)
  • 厘米(cm)
  • 毫米(mm)
  • 四分之一毫米(q)
  • 点(pt)
  • 派卡(pc)
  • 像素(px)

仅当浏览器掌握显示页面的屏幕、所用的打印机或其他用户代理的所有细节时，这些单位才真正有其用处。

对Web浏览器而言，显示效果受屏幕的尺寸和设定的分辨率影响，而创作人员通常对这些因素没有控制力，所以Web页面开发一般不使用绝对长度单位。

但是绝对长度单位在定义文档的印刷样式时非常有用，因为打印机通常使用英寸、点和派卡

像素长度

显示屏上的一个小方块就是一个像素，一般来说，Web浏览器处理font-size: 18px;时基本上会在屏幕上使用指定的像素值，但是对其它设备而言，如打印机，用户代理将根据情况缩放像素长度，即打印代码要确定一像素有几点。

像素度量还常用于表示图像的尺寸，然而对现今主流的响应式设计来说，通常会用相对于文本的尺寸标注图像的大小，而不关心图像中有多少像素。

像素理论

在多数显示器中，每英寸的像素数通常都大于96，有时会大得多。

css规范建议，如果显示器的像素密度与每英寸96像素相差特别大，用户代理应该缩放像素度量，使用”参考像素“。

分辨率单位

随着媒体查询和响应式设计的出现，为了描述显示器的分辨率，出现了三个新单位：

  • 点每英寸(dpi)
  • 点每厘米(dpcm)
  • 点每像素单位(dppx)

相对长度单位

“相对”指其长度相对其他东西而言。

相对长度的实际值（或绝对值）根据不可控的因素而变，比如屏幕分辨率、视区宽度、用户偏好等。

此外，有些相对单位的实际尺寸始终相对当前元素，因此在元素之间也有差异

em和ex单位

1em的值在不同元素之间是不同的，它等于元素的font-size的属性值，如果元素的font-size为14px，那么该元素的1em就等于14px。

1ex的值比1em更复杂，它取决于元素字体中小写字母x的高度，在不同字体中，字母x的高度可能各不相同。

rem单位

与em单位类似，rem也基于声明的字号，但它始终相对根元素计算字号，在HTML文档中，根元素是html。

通常1rem是用户的默认字号，可以通过改变根元素样式改变1rem的值。如果没有为根元素设定字号，font-size: 1rem;的作用与font-size: initial;相同。

ch单位

css3增加的单位，意思是一个字符，css3规范定义它为”等于渲染时所用字体中’0’字形的进距“。

”进距“是css自造的概念，对应字体排印中的”进宽“，为一个字形的起点到下一个字形的起点之间的距离，一般情况下，这个距离等于字形本身的宽度加上侧边的间距（间距可正可负）。

视区相关的单位

css3还增加了三个与视区尺寸相关的单位，它们根据视区的尺寸计算，比如浏览器窗口、可打印区域、移动设备的显示屏等。

  • 视区宽度单位(vw)，1vw等于视区宽度的百分之一。
  • 视区高度单位(vh)，1vh等于视区高度的百分之一。
  • 视区尺寸最小值单位(vmin)，始终取vw或者vh中较小的那个。
  • 视区尺寸最大值单位(vmin)，始终取vw或者vh中较大的那个。

计算值

计算值calc()可以用于进行简单的数学计算，包括加减乘除和括号。

允许在calc()中使用括号似乎是浏览器提供的便利措施，因为calc()的句法定义中没有提到括号，所以浏览器可能会持续支持，但使用时要小心。

能使用calc()计算的地方：<length>、<frequency>、<angle>、<time>、<percentage>、<number>和<integer>。

以上单位类型也能在calc()中使用，但有几个注意点：

  • +和-号两侧的值必须使用相同的单位类型，只有一个例外是<number>和<integer>，计算结果将会是<number>。
  • *计算的两个值中必须有一个是<number>。
  • /计算的除数必须是<number>。
  • 任何情况下除数都不能为0。
  • +和-运算符的两侧必须留有空白，而*和/没有这一限制。
  • 规范要求用户代理至少支持20个算子，算子可以是数字、百分数、长度，超过这一限制的算式被视为无效。

属性值

在一些css属性中，使用attr()可以取得样式对应的元素上的HTML属性值。

结果应该显示为：

[foo]aaaaaaaaaaaaaaaa

[]bbbbbbbbbbbbbbbb

[bar]cccccccccccccccc

但是它几乎没有支持度，所以用不了。

颜色

具名颜色

如果只想使用基本的颜色，最简单的方式是使用颜色的名称，这些颜色被称为具名颜色。

css只有148个具名颜色，数量有限，不能精准满足需求。

RGB和RGBa颜色

计算机中的颜色按不同比例的红绿蓝混合而成，因此经常被称为RGB颜色。

RGB颜色对可得到的颜色几乎没有任何限制。

函数式RGB颜色

有两种颜色值使用RGB表示法，一般句法为rgb(color)，其中color是三个值，可以是百分数或整数，百分数的取值范围是0~100%，整数的取值范围是0~255。

一个颜色值中不能混用整数和百分数，混用后会被视作无效值并被忽略。

使用百分数表示时，有些用户代理可能不支持小数值，会将颜色规整到最近的整数百分数值。

使用整数表示时，就只能使用整数值。

不管使用哪种表示法，超出数值范围的数将被”裁剪“为最近的边界值。

RGBa颜色

从css3起，函数式RGB表示法发展成了函数式RGBa表示法，这种表示法在RGB的三个通道后加了个alpha（透明度）值，

a值的取值范围为0~1的小数，0表示完全透明，1表示完全不透明。超出范围的数将被忽略或裁剪。

十六进制RGB值

css也支持十六进制表示法定义颜色，它把三个00到FF范围的十六进制数串在一起，表示一个颜色。

句法为#RRGGBB，这三个数之间不能有任何空格、逗号或其他分隔符。

因为00~FF转化为十进制即0~255，所以本质上与RGB颜色是等效的。

如果每个十六进制数的两个数字相等，就可以使用简短表示法，比如#BBFFCC等同于#BFC。

十六进制RGBa颜色

在十六进制RGB值后面加一个范围为00~FF的值十六进制值表示alpha，一样可以被简写。

HSL和HSLa颜色

css3引入，HSL是色相、饱和度和明度的简称，句法为hsl(color)，与rgb颜色句法类似。

  • 色相(Hue)，取值范围是0~360（角度值），整数（不是角度类型！）。
  • 饱和度(Saturation)，取值范围是0（无饱和度）~100（完全饱和），百分数。
  • 明度(Lightness)，取值范围是0（全暗）~100（全亮），百分数。

同理，也可以使用范围为0~1的小数alpha值规定透明度。

颜色关键字

有两个特殊的关键字可以在任何允许颜色值的地方使用：

  • transparent，完全透明的颜色，与rgba(0, 0, 0, 0)等效，它也是元素背景色的默认值，还可以用它为元素定义只占空间但不可见的边框，定义渐变时也常用。
  • currentColor，指当前元素的color属性的值。

角度

角度用<angle>表示，即一个<number>后跟下列四个单位中的一个：

  • deg（度数），一圆周有360度。
  • grad（百分度），一圆周有400百分度。
  • rad（弧度），一圆周有2Π。
  • trun（圈数），一圆周有一圈。

角度单位最常在2D和3D变形中使用。

时间和频率

属性的值为一段时间时，使用<time>表示，即一个<number>后跟s（秒）或ms（毫秒）。

时间值最常在过渡、动画、视听css中使用，用于定义持续时间或延迟时间。

在视听css中还有一种值<frequency>，它是一个<number>后跟Hz（赫兹）或kHz（千赫兹），单位不区分大小写。
